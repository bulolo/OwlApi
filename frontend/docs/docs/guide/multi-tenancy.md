# 多租户设计

OwlApi 原生支持多租户隔离，确保不同企业/团队之间的数据完全隔离。

## 隔离策略

| 层级 | 策略 |
| :--- | :--- |
| 数据层 | 所有核心表包含 `tenant_id` 复合主键，查询强制包含租户过滤 |
| 连接层 | gRPC 连接池基于 `tenant_id + runner_id` 索引，确保指令发送到正确节点 |
| 路由层 | 执行路径必须配合 `X-Tenant-ID` 请求头 |

## 身份提取流程

1. **Runner 端** — 启动环境变量 `OWLAPI_TENANT_ID` 决定其归属
2. **API 端** — 从请求头或 JWT 中提取租户 ID，注入 `context.Context` 贯穿整个调用链
